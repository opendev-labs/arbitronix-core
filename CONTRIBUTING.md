# Contributing to Arbitronix Core

Thank you for your interest in contributing to Arbitronix Core! We welcome contributions from the community to make this engine more robust and efficient.

## Our Philosophy
Arbitronix is built for speed, reliability, and quantitative rigor. We value:
- **Asynchronous Code**: All I/O should be non-blocking.
- **Statistical Defensibility**: Strategies must be backed by quantitative logic.
- **Safety First**: Risk management is not an option; it's the core.

## Development Process
1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/amazing-strategy`).
3. Commit your changes with professional, descriptive messages.
4. Push to the branch.
5. Open a Pull Request.

## Coding Standards
- Use `loguru` for all logging.
- Ensure all new features are covered by basic integration tests.
- Follow PEP 8 style guidelines.

## Code of Conduct
Please be respectful and professional in all interactions. We follow the standard Contributor Covenant.
