# Security Policy

## Supported Versions

Only the latest version of Arbitronix Core is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you find a vulnerability, please do NOT open a public issue. Instead:

1. Email us at `security@opendev-labs.example.com` (Placeholder)
2. Provide a detailed description of the vulnerability and steps to reproduce it.

We will acknowledge your report within 48 hours and provide a timeline for a fix.

### Key Vaulting
Never commit your `.env` file or API keys. Always use the provided `.env.example` as a template for local development.
